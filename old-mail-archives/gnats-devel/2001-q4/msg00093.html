>     HG> There are some GNU standards regarding mailing list naming:
> 
>     HG> bug-gnats : To report bugs (really? :-) )
>     HG> help-gnats : Users asking for help (really? :-P )
>     HG> info-gnats : Announcements and progress reports
> 
>     HG> Those are the minimum, I guess we might want to create:
> 
>     HG> gnats-devel : for development discussions.
> 
> What if I wanted the union of help-gnats and gnats-devel instead of two
> separate lists?

Then you should create help-gnats only.  The idea is that for any GNU
software the users of this software know that if they need help, they
do: help-foo, and if they want to report a bug they do: bug-foo.  The
developers?, it's up to them to decide if they want the foo-devel or
not, for some projects there is only one or two maintainers and they
don't need a mailing list.
