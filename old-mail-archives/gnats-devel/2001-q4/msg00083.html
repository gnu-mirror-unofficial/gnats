At 09:47 01.11.2001 -0500, Hugo Gayosso wrote:
>I guess this means that we need to move a GNATS database.
>
>While the current GNATS installation @ gnu.org is using only one
>database, I did some tests using multiple databases and it seems that
>it is working.
>
>Our plan, is/was to have one GNATS database per project, so maybe this
>change will be as easy as defining the new database @ GNU (plus the
>email alias) and then making a tarball of the current database @ Red
>Hat.
>
>Or is it more complex?

Migrating a GNATS 3 database to GNATS 4 on another host should not entail 
more than the following:

- setting up the proper e-mail aliases
- copying the database directory tree
- making an appropriate entry in the databases file
- making whatever changes might be required in the default dbconfig file (I 
think we'd only need to remove a couple of field definitions)
- regenerating the database index with gen-index (GNATS 4 uses a binary format)
- migrating user passwords to MD5 or crypt (I've just finished writing the 
tool gnats-pwconv for this purpose)

In fact, this reminds me that I need to finish the "Upgrading from GNATS 3" 
section of the GNATS 4 manual.
