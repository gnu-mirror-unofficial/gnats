Sorry I didn't get back to you on this one earlier.

I decided not to implement this in Gnatsweb 2 (Unless the release of GNATS 
4 gets pushed even more back now, I consider Gnatsweb 2.x to be dead 
development-wise), but I have it on the list of things that are due to 
appear in Gnatsweb 4. However, Gnatsweb 4 is in beta already, and since 
neither GNATS 3 nor GNATS 4 have mechanisms for marking which fields are 
required in the database config files, I'll take some more time to think 
about how best to implement this.

The fact that Gnatsweb 2 requires such fields as reporter's e-mail, 
submitter-id etc. is actually hardcoded into Gnatsweb and is in no way 
required by GNATS itself. Another important factor is that GNATS 4 is 
extremely configurable, so only a small number of fields are guaranteed to 
exist on all GNATS sites.

I have marked this as a change-request for Gnatsweb 4.1, which I'll start 
work on as soon as GNATS 4 (containing Gnatsweb 4.0) is released, hopefully 
within the next two or three months.
