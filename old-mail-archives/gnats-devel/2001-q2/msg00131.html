Please do not forget those GNATS installations that sit behind a
firewall and cannot get HTTP connections from the outside, but
can get e-mail connections.  E.g., it is relatively easy for me
to get an e-mail address that is reachable from the outside and
can send to the outside.  Though there are, of course, externally
reachable WWW servers, I have severe doubts whether I would be
allowed to run some CGI script there *that contacts the
intranet*.

So I do need an e-mail based frontend.


BTW, concerning anonymous connections to the database: here, too,
reports sent in (and updated!) via e-mail must be kept in mind
(though connecting as "anonymous" with the sender's e-mail
address as password should not be too much trouble, as long as
it can be done non-interactively).
