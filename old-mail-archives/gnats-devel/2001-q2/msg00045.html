Multiple database support semi-broken

  • To: gnats-devel at sources dot redhat dot com
  • Subject: Multiple database support semi-broken
  • From: Yngve Svendsen <yngve dot svendsen at clustra dot com>
  • Date: Mon, 23 Apr 2001 15:56:42 +0200
The current Gnats 4 does not support multiple databases correctly. As the 
transcript from the Gnats session below shows, the CHDB command must be 
issued twice before gnatsd changes database. The default database in the 
Gnats installation was "helpdesk".

Among other things, this breaks multiple database support in Gnatsweb.
