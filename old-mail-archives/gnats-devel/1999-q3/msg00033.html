I should have announced the GNATS database I set up for gnats, but I
never got around to it.  Sorry about that everyone!  Some sneaky fellow
discovered the database and started submitting PRs, and then Rick sneakily
found it and answered the PRs. :-)

Here is all the pertinent information:

  All access to the GNATS gnats db is through gnatsd.  Right now, only
  I have edit access; I'll be happy to add edit access for anyone who
  wants to pitch in.

  The default UI is gnatsweb.  The URL to get in as the read-only guest
  acct is

http://sourceware.cygnus.com/cgi-bin/gnatsweb.pl?database=gnats&user=guest&password=guest&cmd=login

  You can use other UIs if you prefer; tkgnats, the command-line nquery-pr
  et al, and the emacs gnats mode are all popular.

  The e-mail address for submitting new PRs (if someone
  wanted to do it via e-mail instead of using the web UI) is
  gnats-gnats@sourceware.cygnus.com.  This is also the address that
  you should Cc: on _any_ replies or communication related to the PR.
  We used to live by GNATS here at Cygnus, and the rule of thumb was
  that if it wasn't recorded in the PR, it didn't happen.

  I know that "gnats-bugs@sourceware" would have been a more obvious
  address, but I wanted to keep the address consistent for all the
  projects hosted on sourceware.cygnus.com, and "-bugs" is a common name
  used for a discussion mailing list.  Everything on sourceware uses
  "-gnats", e.g. "java-gnats", "autoconf-gnats", and so on.

  There is a new mailing list, "gnats-prs@sourceware.cygnus.com"; all
  new PRs, PR changes, and notes added to PRs are Cc:'ed to this address.
  If you want to keep up with changes in GNATS PRs, this is the mailing
  list to get on.  There is a web archive.  There is a digest.  It works
  like every other mailing list on sourceware.cygnus.com, i.e. send a
  note to gnats-prs-subscribe@sourceware to get on the list.


I'll add links and notes about all of this on the GNATS web page 
(http://sourceware.cygnus.com/gnats/) a little later today.

If front-end authors would like a category for their front-ends, I'll
be happy to set it up.
