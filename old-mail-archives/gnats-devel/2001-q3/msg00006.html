When people create new PR's I get the problem that GNATS produces
garbled mail addresses. It seems that these addresses are the
concatenation of:
- the principal contact mentioned in the submitters file
- the responsible person from the PR's category
- any mail adresses appearing in the notify fields

for example:
from "submitters"
drexler:Hans Drexler:none:1:drexler:tvngeo:

from the file "categories"
WOZteam:GeoTax WOZ team client:bbmgr:tvngeo,maxim,helpdesk

Then a PR filed for category WOZteam send by committer "drexler" causes
an e-mail to be sent to:

drexlertvngeotvngeo (invalid e-mail address)
tvngeotvngeo (invalid e-mail address)

(below is an exerpt from the mail log file).

Does somebody have a clue what is happening here?

We use gnats 3.113 on FreeBSD and sendmail. All names have local mail
accounts.
