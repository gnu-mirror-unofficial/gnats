> Are your changes based upon the current 4.0 branch?  The structure of the
> database is already well defined through the field-config file...?

My changes are based on a CVS checkout from a week ago.  The reason I 
want to expose this information through the daemon is that I want to be 
able to just point my web front end at the daemon and say "Learn!" and 
have it configure itself appropriately.  This way it can run from a 
completely different machine if desired.  It also means that internally 
gnats can move away from the flat file system and move to databases and 
still provide a consistant front end for getting information about the 
structure of the ticket system.

The second step of being able to modify the structure of the database is 
simply an extension to the first idea:  Gnats can be very easy to setup 
and get running in a basic configuration, but can be confusing to get all 
of the flat files right.  I would like to make it so that a user with 
appropriate priviledges can change the configuration of the system.  This 
would also be configurable through one of the config files, to help 
prevent against hacking. =)
