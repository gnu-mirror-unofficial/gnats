The contrib stuff quickly is outdated. I would much rather see a simple
document for each contrib package listing the basic info like web site,
ftp address, mailing lists, cvs access, etc. This will avoid the
current:

DL the gnats tarball, including tkgnats, gnatsweb, etc.
DL the updated tkgnats tar ball ...
DL the updated gnatsweb tarball ...

And for packagers such as myself who build RPMs of these there is no
good reason for the gnats RPM to contain tkgnats, as the tkgnats RPM is
newer anyway.

Please consider removing all contrib except docs listing them from the
gnats tar ball.
