I want to move 'contrib' directory up a level

  • To: gnats-devel at sourceware dot cygnus dot com
  • Subject: I want to move 'contrib' directory up a level
  • From: Jason Molenda <jsm at cygnus dot com>
  • Date: Tue, 26 Oct 1999 09:14:06 -0700
In a gnats release/snapshot, there is the top-level directory
(e.g. ``gnats-3.112.1''), below that is a gnats subdirectory (``gnats''),
and below that is the contrib directory with contributed front-ends like
gnatsweb and tkgnats.

I want to move this contrib directory up a level.  Its presence in the
gnats subdir is a reflection on gnats' history as a Cygnus internal
development project; now that it is a project on its own, there is no
reason to keep that baggage.  I think it will be more obvious for users
to see files organized like

  gnats-3.112.1/gnats
  gnats-3.112.1/send-pr
  gnats-3.112.1/contrib


If no one speaks up, I will do it later this week.  Unfortunately,
cvs does not revision control files or directories, so when I do this
everyone will need to do a "cvs update -d" to update their repositories.
If you check out an old branch (e.g. gnats-3.110) from CVS, the contrib
directory will be in its new location, not its old one as you'd expect.


Jason
