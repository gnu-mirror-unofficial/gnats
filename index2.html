What is it?

GNATS is a freely redistributable set of tools for tracking bug reports. It is one of the most robust bug tracking systems on the net, and is in production use by many companies and free software projects.

There is a lot of confusion about the status of GNATS, and version numbering has been somewhat unclear. There was a version 3.2 release in 1993, but GNATS 3.2 is really broken, so don't use it. Work continued on versions numbered 3.1xx, and many major advances have been made to GNATS in the last couple of years, including multiple database support, customized fields, and comprehensive WWW and Tk based front-ends.

The newest GNATS release is 3.113.1, and the next official release of GNATS will be version 4.0. After a lull in activity during the year 2000, development efforts have picked up again. The 4.0 documentation is nearly done, but some cleanup and bugfixing in the code itself remains. GNATS 4.0 has been through two alpha releases in 1999, but since then a lot of changes have been made, so we probably need another alpha version. A solid beta version is expected in late fall 2001.

Several third-party interfaces and other tools that work with GNATS have been contributed, the most actively developed are Gnatsweb, a web-based interface, and TkGnats, a Tcl/Tk-based interface for UNIX/Linux and Windows. These can be found in the contrib subdirectory of the GNATS distribution. Releases of these tools aren't necessarily synchronized with GNATS releases – see the section on Availability below for details on how to get the newest version.

If you'd like to hear about GNATS announcements, subscribe to the gnats-announce mailing list.

Availability

Released versions

The most recent released versions of GNATS are available from ftp://sources.redhat.com:/pub/gnats/snapshots/.

Contributed software such as Gnatsweb is available at ftp://sources.redhat.com/pub/gnats/contrib/.

The last "official" version was gnats-3.2, but this version has many bugs. We strongly recommend that you use version 3.113.1 instead.

Development versions

The code of what will become GNATS 4.0 is currently only available through CVS. Get it by doing the following:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/gnats login
{enter "anoncvs" as the password}
cvs -z9 -d :pserver:anoncvs@sources.redhat.com:/cvs/gnats co -P gnats

From time to time, bug fixes to the GNATS 3 branch are contributed. These fixes are committed to the CVS repository, even though no further version 3 releases are planned. You can get a snapshot of this as follows:

cvs -d :pserver:anoncvs@sources.redhat.com:/cvs/gnats login
{enter "anoncvs" as the password}
cvs -z9 -d :pserver:anoncvs@sources.redhat.com:/cvs/gnats co -r gnats-v3-branch gnats

The code repository is also accessible via the Web using cvsweb.

Gnatsweb development is hosted in a separate CVS repository at gnu.org. Gnatsweb has two development branches, one for version 2 (which works with GNATS 3.x) and one for version 4 (which works with GNATS 4).

Get the current Gnatsweb 4 development code as follows:

cvs -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/gnatsweb login
(press "enter" when prompted for password)
cvs -z3 -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/gnatsweb co gnatsweb

Get the current Gnatsweb 2 development code as follows:

cvs -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/gnatsweb login
(press "enter" when prompted for password)
cvs -z3 -d:pserver:anoncvs@subversions.gnu.org:/cvsroot/gnatsweb co -r gnatsweb-v2-branch gnatsweb

GNATS mailing lists

There are five GNATS mailing lists:

  • gnats-announce – GNATS-related announcements.
  • gnats-devel – GNATS development discussion.
  • gnats-prs – Changes to GNATS problem reports are recorded here.
  • gnats-cvs – GNATS CVS repository related messages.
  • bug-gnats (hosted at gnu.org)

Here is a handy little widget to subscribe/unsubscribe from the four lists hosted at sources.redhat.com:

Mailing list:
Your e-mail address:
Digest version: (receive list mail batched in a daily digest)
Action:
 

To subscribe to bug-gnats, send a message to bug-gnats-request@gnu.org with a message body of "subscribe bug-gnats''.

You can send mail to any of these lists @sources.redhat.com, e.g. gnats-devel@sources.redhat.com, with the exception of bug-gnats which is hosted at the FSF. Send mail to bug-gnats@gnu.org in that one case.

Reporting GNATS bugs

GNATS uses (gasp) GNATS to track problem reports. We've got the Gnatsweb web interface set up so you can browse the database or submit new problem reports. You can also try out the Gnatsweb interface if you want to see how things look, but please do not submit reports that aren't real. This database is being used by GNATS developers, and is not meant for testing purposes.

All PR transactions/additions/changes are recorded on the gnats-prs@sourceware.cygnus.com mailing list, which has an associated web archive.

Getting involved

We are always on the lookout for people willing to contribute to the project. We need more active developers. Besides, patches for existing problems, contributions to documentation, answers to frequently asked questions or ideas for future development are all very welcome. And, if you aren't a developer or feel able to write good documentation, there is still something you can do: Simply use GNATS and report any bugs and quirks you may find.

GNATS

The current maintainer of GNATS itself is Milan Zamazal. The immediate goal is to get a usable version of GNATS 4 out of the door. This involves making it more robust and updating the documentation. Since development of GNATS lapsed in early 2000, a number of patches have been submitted. These will be applied, and any patches that haven't been submitted due to the uncertain status of GNATS should be sent to the gnats-dev mailing list. Still, the unreleased version 4 requires more work, and people interested in contributing, whether as documentation authors or as developers, should get in touch with him.

Gnatsweb

The current maintainers of Gnatsweb are Yngve Svendsen and Gerald Pfeifer. Several Gnatsweb 2.x releases have been made in the past 6 months, fixing bugs and improving usability and functionality. The 2.x branch is however purely in maintenance mode now, and no further releases are planned unless bugs of earth-shattering importance surface. Focus is now on version 4 of Gnatsweb, which will work only with GNATS 4. A beta release of Gnatsweb 4 is available as part of the current CVS version of GNATS 4. See the section above on availability of development versions for instructions on how to obtain the latest development snapshot of Gnatsweb 4. Anyone interested in participating in Gnatsweb development either by submitting patches or taking directly part in planning and development should get in touch with Yngve or Gerald. Patches should in general be sent to the gnats-dev mailing list.

How to produce patches

Even though people may have good coding skills, they may never have submitted patches to projects such as this before. Following is a quick how-to on producing a patch file.

Check out the GNATS version you intend to modify from the CVS repository, then make the changes you intend to. To produce the patchfile, use the following command:

cvs diff -up file1 file2 ... > patchfile.out

where file1, file2 etc. are the paths to the files that you changed in the source tree. This creates a patch file patchfile.out which contains the differences between the source files in the CVS repository and the checked out files you edited. Since current development efforts are directed at getting version 4 out of the door, patches against the latest development version will be most useful. Patches should go to the gnats-dev mailing list.

Full CVS documentation is available at http://www.loria.fr/~molli/cvs/doc/cvs_toc.html.
